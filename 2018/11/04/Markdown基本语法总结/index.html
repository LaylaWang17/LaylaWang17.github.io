Markdown基本语法总结

2018-11-04

看了很多 Markdown 简明教程，感觉使用 Markdown 最大的优势就是可以完全使用键盘完成排版。但是这个听起来没有什么优势啊，那想象一下传统的排版方式，比如我们想给一段文字加粗，那就需要选择段落、右键、点击加粗按钮。但是在 Markdown 里只需要在这段文字的首尾各加上两个 ** 号就可以了，是不是灰常方便！对于更负责的排版操作，比如标题、列表来说就更是 so easy 啦！

So，为了练习&记忆，自己来整理一下

首先我们需要一个标题

标题是分隔文章层次的好方法，因此 Markdown 也为我们提供了 6 级标题以供使用，而且用法非常简单，只要在文字前加上 # 就可以了。

标题的层级与 # 的数目对应。

（一个温馨提示：# 与文字间需要加上一个空格~）

文字的加粗与斜体

写文章总是要划重点的，因此加粗和斜体必不可少。

  • 加粗：文字前后各加两个 ** 号

  • 斜体：文字前后各加一个 * 号或一个_

我是愉快的分割线

有时候我们需要利用分割线划分文字的内容，在 Markdown 里可以输入---再回车，就会出现一条愉快的分割线啦！

加入一个列表

为了清楚的记录关键点，我们常常使用列表。在 Markdown 里，无序列表直接用*或者-标记就好。

对于有序列表直接敲数字编号。

插入链接和图片

为了增加文章的可读性或者进行一些补充说明时，我们往往需要插入链接和图片，插入的语法为：

  • 链接：[需要显示的文字]（链接地址）

  • 图片：![图题](图片地址)

对于本地图片我们可以使用 新浪图床 等插件快速将获得图片的 Markdown 链接。

（一个温馨提示：插入图片时不要漏掉 ! 哦~）

引用好词好句

写文章时难免需要引用，加个""当然是一种方式，但是看起来不够美观，更好的方式是使用 >，即在引用的段落前加一个 >。

这样被引用的段落会和其他段落直接有明显的区分，而且很好看！

（一个温馨提示：> 与文字间需要加上一个空格~）

插入一段代码

作为一个程序员…写技术类文档时直接插入代码比仅使用文字更直观。

  • 行内代码：将代码部分用反引号括起来。
